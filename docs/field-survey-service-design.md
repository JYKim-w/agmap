|---------|------|---------|------|

```
        ├── 조사원 4
        └── 조사원 5
```

|------|----------|----------|----------|
| IT 숙련도 | 높음 | 중간 | 낮음 (비전문가) |
| 환경 | PC (사무실) | PC (사무실) | 스마트폰 (야외) |
| 인원 | 소수 | 읍면동당 1~2명 | 약 5,000명 |

## 2. 현장조사 배경 (서울경제 기사 기반)

### 조사 개요
- **목적**: 농지 투기 근절, 농지 이용 질서 확립
- **규모**: 전국 농지 약 195만ha
- **인력**: 약 5,000명 (지자체 + 농산물품질관리원 신규 채용)
- **예산**: 1단계 ~670억원, 전체 ~1,100억원
- **기간**: 1단계 2026.05~12 (115만ha), 2단계 2027 (80만ha)

### 조사 항목
| 항목 | 내용 |
|------|------|
| 소유 관계 | 소유자, 취득 경위, 농업인 여부 |
| 실경작 여부 | 실제 농사짓고 있는지, 위탁 영농인지 |
| 휴경 여부 | 무단 휴경, 휴경 기간, 사유 |
| 불법 전용 | 농지를 허가 없이 다른 용도로 사용 |
| 불법 임대차 | 비농업인 소유 후 불법 임대 |
| 시설물 현황 | 비농업 시설, 무허가 건축물 |

### 10대 투기 위험군 (우선 조사 대상)
1. 토지거래허가구역 내 농지
2. 수도권 전역 농지
3. 경매 취득 농지
4. 외지인 취득 농지
5. 공유(공동소유) 취득 농지
6. 농업법인 소유 농지
7. 외국인 소유 농지
8. (기타 3개 유형 — 추정: 상속 미경작, 장기 휴경, 전용 의심)

### 조사 후 조치
- **경미**: 계도 (합법 전환 유도)
- **위반 확인**: 처분 명령 (원상복구, 매각, 이행강제금)
- **투기 위험군**: 유예 없는 즉각 처분

## 3. 업무 프로세스 설계

### 전체 흐름

```
[사전 준비]                    [현장 조사]                    [사후 처리]
    │                              │                              │
    ▼                              ▼                              ▼
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ ① 대상  │→│ ② 조사  │→│ ③ 현장  │→│ ④ 결과  │→│ ⑤ 결과  │→│ ⑥ 통계  │
│   선별  │  │   배정  │  │   실사  │  │   검수  │  │   이관  │  │  보고   │
└─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘
   Web          Web        Mobile       Web            Web          Web
 최종관리자   중간관리자   현장조사자   중간→최종       최종관리자   최종관리자
(농어촌공사) (지자체)     (외주/알바)  (1차→최종컨펌)  (농어촌공사) (농어촌공사)
                                                     ※ 행정처분은
                                                       행정망에서 처리
```

### 단계별 상세

#### ① 대상 선별 (Web — 최종관리자)

```
입력 소스:
  - 농지대장 + 행정 DB (소유자, 취득유형, 지목 등)
  - 드론 정사영상 / 위성영상 (육안 판독 참고)
  - 이전 조사 이력 (기존 위반 건, 변화 여부)
  ↓
최종관리자(농어촌공사)가:
  1. 투기 위험군 조건 필터링 (외지인, 경매, 법인 등)
  2. 정사영상으로 현황 육안 확인 (비농업 이용 의심 여부)
  3. 위험도 등급 부여 (상/중/하)
  4. 조사 대상 최종 확정 / 제외 처리
  5. 읍면동 단위로 지자체에 배분
  ↓
출력: 읍면동별 조사 대상 필지 목록 + 위험도 등급
```

- 위험도 **상**: 불법 전용 의심 (영상에서 건물/시설 확인)
- 위험도 **중**: 휴경 의심, 외지인 소유 등
- 위험도 **하**: 정기 점검 대상

> **향후 확장**: 대상 선별 과정에 AI 영상 판독, 자동 위험도 분류 등을
> 플러그인 형태로 추가할 수 있도록 설계.
> 현재는 관리자 수동 선별 기준으로 구현하되,
> "선별 사유" 필드를 두어 수동 입력이든 AI 판정이든 동일하게 기록.

#### ② 조사 배정 (Web — 중간관리자)

```
중간관리자(지자체 담당자)가:
  1. 최종관리자가 배분한 읍면동 내 필지 목록 확인
  2. 조사원별 담당 필지/구역 배정
  3. 필지별 우선순위 확인 (위험도 등급 기준)
  4. 조사 기한 설정
  5. 조사표(조사 항목) 확정
  ↓
조사원 앱에 자동 반영 → Push 알림
```

#### ③ 현장 실사 (Mobile — 현장조사자)

```
현장조사자가:
  1. 오늘의 할당 필지 확인
  2. 필지로 이동 (경로 안내)
  3. 현장 도착 → 정사영상과 실제 비교
  4. 조사 항목 입력
     - 실경작 여부 (작물 종류, 상태)
     - 휴경 여부 (기간, 사유)
     - 시설물 현황 (유형, 허가 여부)
     - 불법 전용 여부 (용도, 규모)
     - 소유자 접촉 여부, 현장 메모
  5. 증빙 사진 촬영 (전경, 근경, 시설물 등)
  6. 저장 → 다음 필지
```

#### ④ 결과 검수 (Web — 중간관리자 → 최종관리자)

```
[1차 검수] 중간관리자(지자체 담당자)가:
  1. 조사 결과 접수 확인
  2. 데이터 검수
     - 사진 품질 확인
     - 필수 항목 누락 확인
     - 판단 근거 적정성 검토
     - 정사영상 대비 현장 사진 대조
  3. 승인 / 반려 (재조사 요청) / 보완 요청
  4. 반려 시 사유 기재 → 조사원 알림
  ↓
  1차 승인 결과에 따라 분기:
    - "정상" 판단 건 → 자동 확정 (최종관리자 별도 컨펌 불필요)
    - "위반" 판단 건 → 최종 컨펌 단계로 이관
  ↓
[최종 컨펌 — 위반 건만] 최종관리자(농어촌공사)가:
  1. 위반 판단 건만 검토
  2. 위반 유형/수준 최종 판단
  3. 최종 승인 → 결과 이관 단계로 이관
  4. 또는 반려 → 중간관리자에게 반환 (재검토 요청)
```

#### ⑤ 결과 이관 (Web — 최종관리자)

> 행정처분은 정부 행정망(새올시스템, 농지관리시스템 등)에서 처리.
> Agmap의 역할은 **조사 결과 데이터를 만들어서 내보내는 것**까지.

```
최종 컨펌 완료 건에 대해:
  1. 위반 유형 분류 태깅
     - 무단 전용 / 무단 휴경 / 불법 임대차 / 경미(계도)
  2. 이관 데이터 생성
     - 조사 결과 + 증빙 사진 + 정사영상 + 판단 근거
  3. 내보내기
     - 엑셀/CSV 다운로드 (행정망에 수동 입력 시)
     - 표준 포맷 파일 생성 (행정망 일괄 등록 시)
  4. 이관 상태 기록
     - 미이관 / 이관완료 / 처분완료(행정망에서 확인 후 수동 갱신)
```

> **향후 확장**: 행정망 연계 API가 열리면 자동 이관 가능.
> 현재는 내보내기(엑셀/파일) 기반으로 구현하되,
> `export_status` 필드를 두어 이관 이력을 추적.

#### ⑥ 통계/보고 (Web — 역할별)

```
최종관리자(농어촌공사):
  - 시군구/시도 단위 조사 진행률
  - 위반 유형별 전체 통계
  - 지자체별 실적 비교
  - 상급기관 보고서 생성 (농식품부)
  - 1단계/2단계 진행 현황

중간관리자(지자체):
  - 담당 읍면동 내 진행률
  - 조사원별 실적
  - 검수 현황 (대기/승인/반려)
  - 읍면동 단위 보고서
```

## 4. Web 기능 설계 (관리자용)

> 최종관리자 / 중간관리자 모두 Web 사용.
> 역할에 따라 보이는 메뉴와 데이터 범위가 다름.

### 4-0. 역할별 접근 범위

| 기능 | 최종관리자 (농어촌공사) | 중간관리자 (지자체) |
|------|---------------------|-------------------|
| 대상 선별 | 전체 필지 선별·확정 | 조회만 (배분받은 읍면동) |
| 조사 배정 | 읍면동 → 지자체 배분 | 조사원에게 필지 배정 |
| 모니터링 | 전국/시군구 단위 | 담당 읍면동 내 |
| 검수 | 최종 컨펌 | 1차 검수 (승인/반려) |
| 결과 이관 | 위반 태깅, 내보내기 | 조회만 |
| GIS 지도 | 전체 레이어 (위험도, 위반, 진행률) | 담당 읍면동 (상태, 조사원 위치) |
| 통계/보고 | 전국 통계, 상급기관 보고 | 읍면동 통계, 조사원 실적 |
| 시스템 관리 | 전체 사용자·업체·코드 | 담당 조사원 조회 |

### 4-1. 대시보드

| 기능 | 설명 | 역할 |
|------|------|------|
| 전체 현황 | 전체 대상 필지 수, 조사 완료율, 위반 건수 | 공통 (범위 다름) |
| 지역별 진행률 | 시도 → 시군구 → 읍면동 드릴다운 | 최종: 전국 / 중간: 읍면동 |
| 위험도별 현황 | 상/중/하 등급별 완료율 | 공통 |
| 조사원 현황 | 인원, 활성 인원, 일평균 처리 건수 | 중간: 내 조사원 / 최종: 전체 |
| 검수 알림 | 검수 대기 건, 기한 임박 구역 | 공통 |

### 4-2. 대상 선별 관리 (최종관리자)

| 기능 | 설명 |
|------|------|
| 필지 목록/지도 | 전체 농지 목록 + 지도 표시 (GIS 뷰어) |
| 조건 필터링 | 투기 위험군 10대 유형, 지역, 소유 유형 등 다중 필터 |
| 정사영상 확인 | 드론/위성 영상 위에서 필지 현황 육안 확인 |
| 위험도 부여 | 필지별 위험도 등급 설정 (상/중/하) |
| 선별 사유 기록 | 왜 조사 대상인지 사유 기록 (텍스트/코드) |
| 대상 확정 | 조사 대상 최종 확정 / 제외 처리 |
| 일괄 처리 | 조건 기반 일괄 선별 (예: "수도권 외지인 전부 대상") |
| 읍면동 배분 | 확정된 필지를 읍면동 단위로 지자체에 배분 |
| 엑셀 임포트 | 행정망에서 정리된 대상 목록 일괄 등록 |
| 이전 이력 조회 | 해당 필지 과거 조사/처분 이력 |

> **AI 확장 지점**: 선별 사유에 "AI 판정" 유형 추가,
> 위험도 자동 부여 API 연동, 일괄 선별에 AI 추천 목록 반영.
> 현재 수동 워크플로우와 동일한 데이터 구조를 사용하므로
> AI 도입 시 UI 변경 없이 데이터 소스만 추가하면 됨.

### 4-3. 조사 배정 (중간관리자)

| 기능 | 설명 |
|------|------|
| 배분 필지 확인 | 최종관리자가 배분한 읍면동 내 필지 목록 |
| 조사원 배정 | 필지/구역 → 조사원 매핑 (드래그 or 자동) |
| 배정 현황 | 조사원별 담당 필지 수 밸런스 확인 |
| 일정 관리 | 조사 기한 설정, 캘린더 뷰 |
| 조사표 관리 | 조사 항목 템플릿 (양식은 하나, 조건부 항목 노출) |

### 5-2. 조사 지도

| 기능 | 설명 |
|------|------|
| 담당 구역 표시 | 내 구역 경계 + 필지별 상태 색상 |
| 정사영상 레이어 | 드론 촬영 결과 오버레이 (있는 구역만) |
| 영상 커버리지 | "이 구역은 2026.06.15 촬영" 정보 표시 |
| 필지 탭 → 조사 | 필지 선택 시 사전 정보 + 조사 시작 |
| 내 위치 추적 | GPS 기반 현재 위치 + 가장 가까운 미조사 필지 |

#### 필지 가시화 (핵심 — 한눈에 현황 파악)

> 조사원이 지도를 보는 순간 "어디를 가야 하고, 어디를 했는지" 즉시 파악해야 함.
> 축척(줌 레벨)에 따라 표현 방식을 달리하여 가시성 확보.

**줌 레벨별 표현**

| 줌 레벨 | 표현 방식 | 설명 |
|--------|---------|------|
| 원거리 (읍면동 전체) | **상태 색상 필지 채움** | 필지 폴리곤을 상태별 색상으로 채움, 개별 구분 불필요 |
| 중거리 (마을 단위) | **상태 마커 + 필지 경계** | 필지 중심점에 상태 아이콘 마커, 경계선 표시 |
| 근거리 (필지 단위) | **사진 썸네일 + 상세 마커** | 조사 완료 필지에 대표 사진 썸네일 표시 |

**상태 마커 아이콘**

| 상태 | 색상 | 마커 아이콘 | 설명 |
|------|------|-----------|------|
| 미조사 | 빨강 | ○ 빈 원 | 아직 조사 안 함 |
| 진행중 | 노랑 | ◐ 반원 | 저장 안 하고 이탈 or 임시저장 |
| 완료 (검수 대기) | 파랑 | ● 채운 원 | 조사 완료, 검수 대기 중 |
| 승인 | 초록 | ✓ 체크 | 검수 통과 |
| 반려 | 주황 | ↻ 화살표 | 재조사 필요 — 눈에 띄어야 함 |
| 위반 확정 | 빨강+깃발 | ⚑ 깃발 | 위반으로 최종 확정 |

**사진 썸네일 (근거리 줌)**

```
┌─────────────┐
│  [필지 경계]  │
│              │
│   ┌──────┐  │
│   │ 📷   │  │  ← 조사 시 촬영한 대표 사진 (전경) 썸네일
│   │ 사진  │  │     미조사 필지에는 표시 안 함
│   └──┬───┘  │
│      ✓      │  ← 상태 아이콘 (승인됨)
└─────────────┘
```

- 조사 완료 필지: 대표 사진(전경) 썸네일을 필지 위에 작게 표시
- 미조사 필지: 썸네일 없이 상태 마커만
- 썸네일 탭 → 사진 확대 + 조사 요약 팝업
- 사진이 여러 장이면 썸네일 우측 상단에 매수 뱃지 (예: "3")

**필지 탭 시 팝업 (간략 정보)**

```
┌────────────────────────────┐
│ 📍 전남 곡성군 삼기면 경악리 123  │
│                            │
│ 상태: ✓ 승인               │
│ 조사일: 2026.06.20         │
│ 판단: 정상 (경작중)          │
│                            │
│ [📷 전경] [📷 근경] [📷 +1]  │  ← 사진 썸네일 가로 나열
│                            │
│ [상세 보기]    [재조사]      │
└────────────────────────────┘
```

**가시성 보조 기능**

| 기능 | 설명 |
|------|------|
| 상태 필터 | "미조사만 보기", "반려만 보기" 등 토글 |
| 클러스터링 | 원거리에서 밀집 필지를 숫자 뱃지로 묶음 (예: "미조사 23건") |
| 반려 강조 | 반려 건은 마커 크기 확대 + 깜빡임 애니메이션 |
| 오늘 할당 강조 | 오늘 배정된 필지에 테두리 강조 (굵은 선) |
| 범례 | 화면 하단에 색상 범례 토글 (첫 사용 시 자동 표시) |

### 5-3. 조사 입력 (핵심 — 재설계)

> 현재 farmfield 양식 → 농지 전수조사 항목으로 전면 개편

#### 필지 기본 정보 (사전 구축 DB에서 로드)

> 행정 DB 연동 또는 행정망에서 정리된 대상 목록을 엑셀로 임포트.
> 데이터 확보가 안 되는 항목은 "미제공"으로 표시. 조사에는 지장 없도록 설계.

| 항목 | 데이터 출처 | 비고 |
|------|-----------|------|

> 사진은 촬영 시 GPS 좌표 + 촬영 방향 + 타임스탬프 자동 기록

### 5-4. 조사 품질 검증 (저장 시 자동)

> 경고 무시 가능하되 무시 사실 기록 → 검수 시 관리자에게 표시

## 8. AI 확장 설계 가이드

> 현재는 AI 없이 관리자 수동으로 운영.
> 아래 지점에 AI를 도입해도 기존 워크플로우가 깨지지 않도록 설계.

### 확장 가능 지점

| 단계 | AI 적용 | 현재 (수동) | AI 도입 시 |
|------|--------|-----------|-----------|
| ① 대상 선별 | 영상 판독 | 관리자가 정사영상 육안 확인 | AI가 의심 필지 자동 추출 → 관리자 확인 |
| ① 대상 선별 | 위험도 분류 | 관리자가 수동 등급 부여 | AI가 자동 등급 제안 → 관리자 확정 |
| ④ 검수 | 이상치 탐지 | 관리자가 일일이 확인 | AI가 의심 건 우선 표시 → 관리자 집중 검수 |
| ⑥ 통계 | 패턴 분석 | 단순 집계 | AI가 위반 패턴 분석, 고위험 지역 예측 |

### 데이터 설계 원칙 (AI 대비)

```
필지 테이블:
  - selection_reason    TEXT    -- 선별 사유 (수동 입력 or "AI: 비농업 시설 감지")
  - selection_source    ENUM    -- 'MANUAL' | 'AI' | 'RULE'  ← 확장용
  - risk_grade          ENUM    -- 상/중/하
  - risk_score          FLOAT   -- NULL (수동 시) | 0.0~1.0 (AI 도입 시)
```

> `selection_source`와 `risk_score` 필드를 미리 두되, 현재는 MANUAL/NULL로 운영.
> AI 도입 시 이 필드만 채우면 되므로 스키마 변경 불필요.

## 10. 기술 고려사항

### 비전문가 UX (★ 최우선)
- 5,000명 중 상당수가 IT 비숙련자
- **조사 폼**: 최대한 선택형 (드롭다운, 버튼), 텍스트 입력 최소화
- **단계별 안내**: 위저드 형태로 항목 순서대로 진행
- **큰 터치 영역**: 야외 장갑/햇빛 환경 고려
- **실수 방지**: 저장 전 요약 확인 화면
- **교육 비용 최소화**: 인앱 가이드, 툴팁

### 대규모 운영
- 동시 접속 5,000명 모바일 + 관리자
- **사진 스토리지**: 필지당 3~5장 × 195만 필지 = 12~29TB 예상
  - 업로드 시 자동 리사이즈 (원본 1200px 이하 압축)
  - 별도 오브젝트 스토리지 (S3 등) 필수
  - 썸네일 사전 생성 (50x50px) — 지도 표시용
- **GIS 렌더링**: 195만 필지 폴리곤을 WFS로 전부 로드하면 브라우저 다운
  - 조사 상태 색상은 WMS 타일로 렌더링 (Geoserver SLD 스타일)
  - WFS는 개별 필지 클릭 조회용으로만 사용
- **오프라인 지도**: 타일 다운로드 시 용량 주의 (읍면동 단위 수백 MB)
  - 줌 레벨 상한선 설정, 정사영상은 오프라인 제외 검토
  - VWorld 타일 오프라인 저장 이용약관 확인 필요
- 오프라인 동기화 충돌 해결

### 데이터 확보
- 필지 기본 정보는 **행정 DB 연동 또는 엑셀 임포트** 두 경로 지원
- 개인정보(소유자명, 거주지 등)는 개인정보보호법 범위 내에서 제공
- 데이터 미확보 항목은 "미제공"으로 표시, 조사 진행에 지장 없도록 설계
- 사진 촬영 방향(compass heading)은 참고용으로만 기록 (디바이스 센서 정확도 한계)

### 인증 / 권한
- 현재: 모바일 `/user/login` (farmfield), Web `/auth/api/login` (JWT)
- 통합 필요: 하나의 인증 체계로 3역할 구분
- 역할: `SUPER_ADMIN` (최종관리자), `ADMIN` (중간관리자), `SURVEYOR` (현장조사자)
- 데이터 범위: 중간관리자는 담당 읍면동만, 현장조사자는 배정받은 필지만
