# Plan: home-survey-ux

> **Feature**: 홈/조사 탭 UI/UX 재설계
> **Phase**: Plan
> **Date**: 2026-04-22
> **Author**: JY Kim

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 홈 탭의 "미완료" 세그먼트와 조사 탭이 동일한 배정 목록을 이중 노출. 조사 위저드 진입점이 2곳 이상 분산되어 사용자가 어디서 시작해야 하는지 혼란. |
| **Solution** | 홈 = 순수 대시보드(KPI·실적·알림), 조사 = 전체 작업 목록(미완료·제출현황). 역할을 명확히 분리하고 진입점을 조사 탭으로 단일화. |
| **UX Effect** | 대시보드에서 현황 파악 → 조사 탭에서 실제 작업 진행. 흐름이 직관적이고 탭 간 중복 제거. |
| **Core Value** | 조사원이 "지금 뭘 해야 하나"를 홈에서 파악하고 조사 탭에서 즉시 실행할 수 있는 명확한 두-스텝 워크플로우. |

---

## Context Anchor

| 항목 | 내용 |
|------|------|
| **WHY** | 홈·조사 탭 기능 중복으로 조사원이 어디서 시작해야 할지 혼란. 단순하고 명확한 워크플로우 필요. |
| **WHO** | 현장 조사원 (모바일 앱 주 사용자). 빠른 업무 처리가 목표. |
| **RISK** | 홈 목록 제거 시 습관적으로 홈에서 조사 시작하던 사용자 혼란 가능. |
| **SUCCESS** | 동일 데이터 중복 노출 0건, 조사 진입점 1곳으로 통일, 탭별 역할 단일화. |
| **SCOPE** | `app/(tabs)/index.tsx`, `app/(tabs)/survey.tsx` 수정. 라우팅·API 변경 없음. |

## 1. 문제 정의

### 현재 탭 구조 (4탭)

| 탭 | 현재 역할 | 문제 |
|----|----------|------|
| 홈 | KPI + 이번주실적 + **미완료 탭**(미조사+DRAFT) + **제출현황 탭** + 검색 | 조사 탭과 목록 중복 |
| 조사 | 임시저장(DRAFT) + 미조사 목록 | 홈 미완료 탭과 동일 |
| 지도 | 필지 지도 + 조사 진입 | 변경 없음 |
| 내정보 | 프로필 + 비밀번호 + 공지사항 | 변경 없음 |

### 중복 상세

- 홈의 `미완료 탭` = 조사 탭 (`!resultId || resultStatus === 'DRAFT'`) — **동일 데이터**
- 조사 위저드 진입점: 홈 카드 탭 / 조사 탭 / 지도 팝업 3곳 분산

---

## 2. 목표 구조

### 재설계 후 탭 구조

| 탭 | 역할 | 주요 내용 |
|----|------|----------|
| **홈** | 대시보드 | 배정현황 KPI · 이번주실적 · 반려 알림 배너 · 핀고정 공지 배너 |
| **조사** | 작업 목록 | 세그먼트: 미완료(미조사+DRAFT) / 제출현황(SUBMITTED~) · 검색 |
| **지도** | 지도 | 변경 없음 |
| **내정보** | 계정관리 | 프로필 · 비밀번호 변경 · 공지사항 전체 목록 |

### 역할 분리 원칙

```
홈 탭          →  "지금 어떤 상황인가?" (Read-only 현황 파악)
조사 탭        →  "지금 무엇을 해야 하나?" (Action 목록 + 진입)
지도 탭        →  "어디에 있는 필지인가?" (공간 시각화 + 조사 진입)
내정보 탭      →  "내 계정과 공지" (Settings)
```

## 3. 요구사항

### 홈 탭 변경 (index.tsx)

| # | 요구사항 | 우선순위 |
|---|---------|---------|
| H-1 | 배정현황 KPI 카드 유지 (ASSIGNED/IN_PROGRESS/COMPLETED/RETURNED + 완료율) | Must |
| H-2 | 이번주 실적 위젯 유지 | Must |
| H-3 | 반려 알림 배너 유지 (반려 N건 → 조사 탭 이동) | Must |
| H-4 | **목록(미완료 탭, 제출현황 탭, 검색바) 제거** | Must |
| H-5 | pinned 공지 배너 추가 (notices 중 `pinned=true` 최신 1건) | Should |
| H-6 | 동기화 대기 배너 유지 | Should |
| H-7 | "조사 시작하기" CTA 버튼 추가 → 조사 탭으로 이동 | Should |

### 조사 탭 변경 (survey.tsx)

| # | 요구사항 | 우선순위 |
|---|---------|---------|
| S-1 | 세그먼트 탭 추가: **미완료** / **제출현황** | Must |
| S-2 | 미완료 탭: 미조사 + DRAFT, 홈에서 이동해온 카드 UI 적용 (D-day, 우선순위, assignStatus 뱃지) | Must |
| S-3 | 제출현황 탭: SUBMITTED/UNDER_REVIEW/APPROVED/REJECTED 목록 + 상단 요약 카운트 | Must |
| S-4 | 검색바 (주소 검색) 추가 | Must |
| S-5 | 반려 건 카드에 "재조사 시작" 버튼 | Must |
| S-6 | 미완료 0건 시 "모든 조사 완료" 빈 상태 유지 | Must |

### 조사 ↔ 지도 인터랙션 (Option C 선택)

| # | 요구사항 | 우선순위 |
|---|---------|---------|
| M-1 | 조사 탭 카드에 "지도" 버튼 추가 → 탭 시 지도 탭으로 전환 + 해당 필지 카메라 포커스 | Must |
| M-2 | 지도 탭 하단 시트(BottomSheet): 배정 목록 항목 표시 (항상 표시, 위로 스와이프 시 확장) | Must |
| M-3 | 하단 시트 카드 탭 → 지도 카메라 해당 필지로 이동 | Must |
| M-4 | 지도 핀/폴리곤 탭 → 하단 시트에서 해당 카드 하이라이트 | Should |
| M-5 | 탭 전환 시 selectedAssignmentId 전역 상태로 공유 (Zustand) | Must |

### 공지 배너 (홈 탭 신규)

| # | 요구사항 | 우선순위 |
|---|---------|---------|
| N-1 | `useNoticesStore`에서 pinned 공지 1건 조회 | Should |

## 5. 성공 기준

| # | 기준 |
|---|------|
| SC-1 | 홈 탭에 배정 목록(FlatList) 없음 — 카드 클릭으로 조사 진입 불가 |
| SC-2 | 조사 탭에 미완료/제출현황 세그먼트 2개 존재 |
| SC-3 | 조사 탭 미완료 탭이 `!resultId || resultStatus === 'DRAFT'` 데이터만 표시 |
| SC-4 | 조사 탭 제출현황 탭이 `SUBMITTED` 이상 + `rejected` 데이터 표시 |
| SC-5 | 홈 탭에 pinned 공지 배너 표시 (notices 있을 경우) |
| SC-6 | 반려 배너 탭 시 조사 탭으로 이동 |
| SC-7 | 조사 탭 카드 "지도" 버튼 탭 → 지도 탭 전환 + 해당 필지 포커스 |
| SC-8 | 지도 탭에 하단 시트 배정 목록 존재 (항상 표시) |
| SC-9 | selectedAssignmentId 상태가 조사↔지도 탭 간 공유됨 |
