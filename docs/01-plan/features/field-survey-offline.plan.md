# Field Survey Offline — 네트워크 불안정 대응 Plan

> 작성일: 2026-04-05
> 선행 PDCA: field-survey (1차 사이클 완료)

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 네트워크 불안정 시 조사 제출 실패 → 데이터 유실 위험 |
| **Solution** | 제출 실패 시 로컬 큐 저장 + 네트워크 복귀 시 자동 재시도 + 오프라인 상태 표시 |
| **UX Effect** | 조사원이 네트워크 상태를 신경 쓰지 않고 조사에 집중 |
| **Core Value** | 통신 불안정 환경에서도 데이터 유실 없는 안정적 조사 플로우 |

## Context Anchor

| Key | Value |
|-----|-------|
| **WHY** | 농촌/산간 현장에서 일시적 통신 불가 시 제출 실패 → 재작업 부담 |
| **WHO** | 현장 조사원 5,000명 (농촌 환경) |
| **RISK** | AsyncStorage 용량 한계, 사진 파일 대용량, 동기화 충돌 |
| **SUCCESS** | 네트워크 끊김 상태에서 조사 완료 → 복귀 시 자동 제출 → 데이터 유실 0건 |
| **SCOPE** | 제출 큐 + 사진 큐 + 네트워크 감지 + 오프라인 배너 |

## 1. 현재 상태

| 항목 | 현재 동작 | 문제 |
|------|----------|------|
| 조사 데이터 | AsyncStorage 자동 임시저장 | 제출 시 네트워크 필요 — 실패하면 에러만 표시 |
| 사진 | 제출 성공 후 순차 업로드 | 제출 성공 + 사진 실패 시 불완전한 데이터 |
| 네트워크 감지 | 없음 | 오프라인인지 사용자가 알 수 없음 |
| 할당 목록 | API 호출 필수 | 오프라인이면 빈 화면 |

## 2. 성공 기준

| # | 기준 | 검증 방법 |
|---|------|----------|
| SC-1 | 네트워크 끊긴 상태에서 조사 제출 시도 → 로컬 큐 저장 + Toast 안내 | 비행기 모드에서 제출 → "오프라인 저장됨" |
| SC-2 | 네트워크 복귀 시 큐에 저장된 건 자동 제출 | WiFi 켬 → 자동 업로드 + Toast "N건 동기화 완료" |
| SC-3 | 사진 업로드 실패 시 재시도 큐 동작 | 사진 업로드 중 네트워크 끊김 → 복귀 후 재시도 |
| SC-4 | 오프라인 상태 배너 표시 | 네트워크 끊기면 화면 상단에 "오프라인" 배너 |
| SC-5 | 할당 목록 캐시 — 오프라인에서도 마지막 조회 목록 표시 | 비행기 모드에서 앱 실행 → 이전 목록 표시 |

## 3. 구현 범위

### 3.1 제출 큐 (Submit Queue)

```
제출 시도
  ├── 온라인 → 서버 API 호출 → 성공
  └── 오프라인 or 실패 → AsyncStorage 큐에 저장
                          ├── {body, photos[], timestamp, retryCount}
                          └── 네트워크 복귀 이벤트 → 큐 순차 처리
```

- `lib/offline/submitQueue.ts` — 큐 관리
- 큐 항목: `{ id, body: SurveyResultInput, photos: PhotoEntry[], createdAt, retryCount }`
- 최대 재시도: 5회
- 재시도 간격: 5초 → 15초 → 30초 (지수 백오프)

### 3.2 사진 업로드 큐 (Photo Queue)

- 제출 성공 후 사진 순차 업로드
- 실패 시 개별 사진 재시도 (제출 건과 독립)
- 로컬 파일 경로 유지 → 업로드 완료 후 정리

### 3.3 네트워크 상태 감지

- `@react-native-community/netinfo` 활용 (이미 설치됨)
- `lib/offline/networkStatus.ts` — 글로벌 네트워크 상태 store
- 상태 변경 이벤트 → 큐 자동 처리 트리거

### 3.4 오프라인 UI

- 오프라인 배너: 화면 상단 "오프라인 — 데이터는 자동 저장됩니다" (warning-light 배경)
- 동기화 대기 배지: 홈 탭에 큐 대기 건수 표시
- 제출 시 오프라인이면: Toast "오프라인 저장됨. 네트워크 복귀 시 자동 제출"

### 3.5 할당 목록 캐시

- API 응답을 AsyncStorage에 캐시
- 오프라인 시 캐시에서 로드
- 온라인 복귀 시 자동 갱신
